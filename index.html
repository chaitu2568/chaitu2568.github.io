Internal Tool

MCP Production Health Monitor

Originated and built an internal Model Context Protocol (MCP) server that automates production service-health reviews across US/EU/APJ regions — replacing what used to be a multi-hour weekly manual ritual with a structured, auditable report generated in minutes. Authored as a full TypeScript multi-agent system on LangGraph.js, with a StateGraph-driven supervisor orchestrating parallel collector, triage, and report-writer agents. Hybrid rendering keeps numbers deterministic (extracted directly from raw telemetry) and prose LLM-polished, with fingerprint-based P1/P2/P3 triage and anti-hallucination guardrails baked into every stage.

  • Correlates Dynatrace metrics, Kibana error logs, PagerDuty incidents, and Kubernetes telemetry into a single actionable report
  • StateGraph supervisor coordinates collector / triage / report-writer agents with structured hand-offs and typed intermediate state
  • Fingerprint-based P1/P2/P3 classification catches recurring incidents; hybrid deterministic + LLM rendering eliminates hallucinated numbers
  • Extended with infrastructure observability — pod counts, CPU/memory, HPA/autoscaling analysis surfaced inside the AI workflow
  • Cut on-call review time from hours to minutes; template for safe agentic AI in a highly regulated production domain
  • Composes with a broader library of 25+ reusable AI skills the team uses daily (escalation triage, log search, incident investigation, PR review, vuln tracking)
